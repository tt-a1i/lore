01 — Why

代码已经廉价，理解没有。当 agent 写下大部分代码，真实的需求、半途发现的约束、被否掉的方案——都活在会话结束即蒸发的对话里。 Code is cheap now; understanding is not. When agents write most of the code, the real requirements, mid-flight constraints, and rejected approaches live in conversations that evaporate when the session ends.

给人For humans

lore why <file>:<line>

从任何一行代码，回溯到产生它的那段对话。审代码不再靠猜测意图。 From any line of code back to the conversation behind it. Review code without guessing intent.

给 agentFor agents

项目的情景记忆Episodic memory for your repo

MCP server 让任何 agent 继承项目历史——过去的决策、被否的方案、每个文件的演化。新会话不再冷启动。 An MCP server gives any agent your project's past decisions, rejected approaches, and file evolution. New sessions stop being cold starts.

02 — How it works

四步，从散落的 transcript 到可追问的图谱Four steps from loose transcripts to a queryable graph

lore 架在 Git 之上——解析磁盘上已有的对话记录，安装当下即可回溯索引全部历史。 lore sits on top of Git — it parses the transcripts already on your disk, retroactively indexing your entire history the minute you install.

03

图谱Graph

Graph

会话、commit、文件与决策进入嵌入式图存储；双时间有效性——被取代的决策标记失效，从不删除。 Sessions, commits, files and decisions in an embedded graph with bi-temporal validity — superseded decisions are invalidated, never deleted.

04

蒸馏Distill

Distill

每会话一次 claude -p 调用，提取决策、约束与被否方案，逐条锚回原始对话消息。 One claude -p call per session extracts decisions, constraints and rejected approaches — each anchored back to the exact messages.

03 — Commands

八个命令，一条认知链路Eight commands, one chain of understanding

lore scan --repo <path>索引 transcript，跑匹配引擎，构建图谱Index transcripts, run the match engine, build the graph
lore why <file>:<line>从任意一行代码回溯到产生它的对话Trace any line back to the conversation that produced it
lore history <file>文件的完整演化时间线，逐 commit 附对话归因Full evolution timeline of a file, with attribution per commit
lore distillLLM 蒸馏会话为决策 / 约束 / 被否方案笔记Distill sessions into Decision / Constraint / RejectedApproach notes
lore ask "<question>"跨决策与对话消息的全文检索Full-text search across decisions and conversation messages
lore sample -n <N>抽样匹配结果，人工或 agent 抽查归因准确率Sample matches for spot-checking attribution accuracy
lore mcp启动 stdio MCP server，给 agent 三个记忆工具Start the stdio MCP server exposing three memory tools
lore serve --port 4017本地图谱 viewer，四视图 + 时间轴回放Local graph viewer — four views plus timeline playback

05 — MCP

给每个 agent 接上项目记忆Plug project memory into any agent

在任何 MCP host 里加一段配置，agent 即获得三个工具：行级归因、决策检索、文件演化。它写代码前，先知道这个仓库经历过什么。 One config block in any MCP host and the agent gains three tools: line-level attribution, decision search, file evolution. Before it writes code, it knows what this repo has been through.

  }
